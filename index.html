Chapter 01 · Getting Started

用 Suno 寫一首歌，
其實沒你想的那麼難

Suno 是一個用「文字描述」生成音樂的 AI 工具。你不需要懂樂理、不用會樂器，只要學會幾個關鍵的「下指令」方式，就能做出完整的歌曲。這份指南會帶你從零開始，重點放在 Style 描述、歌詞結構，以及最常被搞混的 [ ] 與 ( ) 用法。

你會學到

Style 描述PROMPTING

如何用樂器、風格、情緒組合出有效的 Style prompt。

歌詞結構STRUCTURE

主歌、副歌、橋段怎麼安排，AI 才聽得懂。

[ ] 標籤語法SECTION TAGS

告訴 AI「這段是什麼角色」的官方語法。

( ) 演唱註記PERFORMANCE NOTES

寫進歌詞裡的演唱指示，與 [ ] 完全不同。

i
學習建議 這份指南附帶可複製的 prompt 範例與互動式結構模擬器。看到範例就動手試一次，比讀十遍都有用。
Chapter 02 · Workflow

建立一首歌的
三個欄位

當你切換到 Custom（自訂）模式，會看到三個關鍵欄位。理解它們各自負責什麼，是寫好歌的第一步。

Lyrics（歌詞）
放入歌詞文字。可以用 [Verse]、[Chorus] 等標籤標示段落結構。本指南第 6、7 章詳細說明。
Style（風格描述）
用英文描述你想要的音樂風格，例如「acoustic ballad, soft female vocal, piano」。建議 200 字內，描述越精準越好，但太長反而模糊。
Title（標題）
作品名稱，純粹給你管理用，不會大幅影響生成結果。

一個最簡單的範例

下面是一個「最小可運作」的設定，複製過去就能跑：

Style

STYLE PROMPT
acoustic pop ballad, soft female vocal, fingerstyle guitar, gentle piano, warm and nostalgic, 70 bpm

Lyrics

LYRICS
[Verse]
街燈把影子拉長
我獨自走過熟悉的轉角

[Chorus]
愛你像呼吸一樣
不需要理由
✓
核心觀念 Style 描述「聲音聽起來怎樣」，Lyrics 描述「演唱的內容與段落」。兩者互不混用。
Chapter 03 · Style Prompt

Style 欄位的
黃金公式

寫 Style 不用文謅謅，有效的描述通常包含這四個面向：風格、樂器、人聲、氛圍/節奏。把它們用半形逗號串起來就是一個好 prompt。

公式

FORMULA
[風格 genre], [樂器 instruments], [人聲 vocal], [氛圍 mood], [節奏 bpm]

套用範例

EXAMPLE
indie folk, acoustic guitar, light strings, soft male vocal, melancholic and warm, 80 bpm

實用建議

原則說明
用英文Suno 對英文 prompt 反應最準。中文也可以，但精準度較差。
用逗號分隔用半形逗號分開不同關鍵字，AI 會把每個都當成一個獨立指令。
200 字內過長反而會稀釋重點。精煉到 5–8 個關鍵詞最佳。
避免矛盾不要同時寫「heavy metal」和「lullaby」，AI 會混亂。
具體 > 抽象「fingerstyle guitar」比「nice guitar」有用得多。

初學者常用詞庫

人聲 Vocal

柔美女聲 soft female vocal 磁性男聲 deep male vocal 空靈女聲 ethereal female vocal 沙啞男聲 raspy male vocal 合聲 harmonies / choir 無人聲 instrumental

氛圍 Mood

溫暖懷舊 warm, nostalgic 憂傷 melancholic 夢幻 dreamy 熱血 uplifting, energetic 沉靜 calm, peaceful 史詩 epic, cinematic

節奏 BPM 速查

60–75抒情慢板，搖籃曲、情歌的副歌
80–95抒情中板，多數 K 歌主歌速度
100–120流行常用，舒服的律動感
120–140熱鬧、舞曲、City Pop
140+熱血搖滾、電子舞曲、嘻哈快歌

Chapter 06 · Song Structure

一首歌的
「骨架」是什麼

Suno 雖然會自動安排段落，但給它清楚的結構標籤，生成的音樂會更有層次與記憶點。流行歌最常見的結構，就這幾個區塊。

七種基本段落

標籤角色建議用法
[Intro]前奏建立氛圍，可純樂器，2–4 句或留空。
[Verse]主歌故事鋪陳的地方，旋律相對平穩。
[Pre-Chorus]導歌主歌與副歌之間的橋，把情緒推上去。
[Chorus]副歌歌曲記憶點。每次重複歌詞與旋律最好一致。
[Bridge]橋段整首歌的轉折，通常出現在第二副歌之後。
[Instrumental]間奏純樂器段落，常用於 solo 或氣氛切換。
[Outro]尾奏收尾，可以漸弱、可以重複關鍵句。

Chapter 07 · Bracket Syntax

[ ] 與 ( )
看似相像，作用完全不同

這是初學者最常搞混的地方。方括號 [ ] 是給 AI 看的「結構指令」，圓括號 ( ) 是模擬真人歌詞中的「演唱註記」。一個寫錯，整首歌的味道就跑掉了。

[ ] 方括號SECTION TAG

用途：告訴 AI「這段是什麼角色」

位置：單獨一行，段落開頭

會被唱出來嗎？不會。AI 只把它當成結構指令。

( ) 圓括號PERFORMANCE NOTE

用途：標示和聲、回應、轉音、氣音等演唱表現

位置：寫在歌詞句子裡

會被唱出來嗎？會。通常以較輕、較弱、或合聲方式唱出。

[ ] 方括號的常見用法

1. 段落結構標籤

SECTION TAGS
[Verse 1]
[Pre-Chorus]
[Chorus]
[Bridge]
[Outro]

2. 段落內加屬性（進階）

可以在 [ ] 裡加上人聲或情緒指示，AI 會盡量遵循：

ADVANCED TAGS
[Verse 1: female vocal, soft]
[Chorus: powerful, full band]
[Bridge: a cappella]
[Instrumental: guitar solo]

3. 純效果指令

EFFECT TAGS
[Build up]     # 醞釀、推進
[Drop]         # 副歌爆發點
[Fade out]     # 結尾漸弱
[End]          # 歌曲結束

( ) 圓括號的常見用法

圓括號裡的字會被「唱出來」，但通常是和聲、回應、或氣音的方式。最常見的四種：

用途範例聽起來像
呼應 / 和聲我愛你 (我愛你)主唱唱完，合聲輕聲重複
氣音 / 哼唱(oooh) (ahhh) (la la la)無歌詞的哼唱
背景補充詞別走 (請別走)背景人聲補一句
演唱情境註記(whispered) (漸弱)提示這段用氣音 / 漸弱

同一段歌詞，兩種寫法的差別

切換下方的 Tab 比較兩種寫法生成出來的「感覺」差異：

VERSION A — 純歌詞
愛你像呼吸一樣
不需要理由
每一秒都是奇蹟
每一刻都剛好

AI 會自己決定段落、和聲、強弱。可能可以、也可能平淡。

VERSION B — 加圓括號
愛你像呼吸一樣 (像呼吸一樣)
不需要理由 (oh, no reason)
每一秒都是奇蹟 (yeah)
每一刻都剛好 (just right)

圓括號會被當成和聲或回應唱出來。明顯多了「合唱層次」與西式 R&B 的味道。

VERSION C — 加方括號
[Chorus: powerful, full band]
愛你像呼吸一樣
不需要理由
每一秒都是奇蹟
每一刻都剛好

[ ] 不會被唱出來，但會明確指引這段是「副歌、有力、全編制」，使整段更有副歌的爆發感。

進階：兩者組合的最佳姿勢

實務上，最強的寫法是「段落用 [ ]、句子內用 ( )」兩者一起用：

BEST PRACTICE — COMBINED
[Chorus: powerful, female lead with male harmony]
愛你像呼吸一樣 (像呼吸一樣)
不需要理由 (no reason at all)
每一秒都是奇蹟 (yeah, yeah)
每一刻都剛好 (oh oh oh)
✓
記住這個原則就夠 [ ] 是給 AI 看的「指令」、不會被唱出來；( ) 是寫進歌詞裡的「演唱動作」、會被輕聲唱出來。

Chapter 09 · Common Mistakes

初學者最常踩的
5 個雷

這些錯誤的特色是：技術上不會報錯，但會讓 AI 完全會錯意，生出來的歌怪怪的卻說不出哪裡怪。

錯誤 1 · 把段落標籤寫進歌詞行內

❌ 錯誤

WRONG
[Verse] 街燈把影子拉長
我獨自走過熟悉的轉角

✓ 正確

RIGHT
[Verse]
街燈把影子拉長
我獨自走過熟悉的轉角

[ ] 標籤永遠單獨一行，下一行才是歌詞。

錯誤 2 · 把樂器寫進歌詞欄

❌ 錯誤（寫在 Lyrics 欄）

LYRICS FIELD
piano, soft female vocal,
warm and nostalgic
[Verse]
街燈把影子拉長

✓ 正確（樂器寫 Style 欄）

STYLE FIELD
piano, soft female vocal,
warm and nostalgic, 75 bpm

樂器、風格、BPM 全都屬於 Style 欄；Lyrics 欄只放歌詞與 [ ] / ( ) 標籤。

錯誤 3 · ( ) 與 [ ] 互相亂用

❌ 錯誤

WRONG
(Chorus)
愛你像呼吸一樣 [像呼吸一樣]

段落用 ( ) 不會生效；和聲用 [ ] 反而可能被當成奇怪的指令。

✓ 正確

RIGHT
[Chorus]
愛你像呼吸一樣 (像呼吸一樣)

錯誤 4 · Style 寫得太貪心

❌ 太多互相打架的詞

WRONG
pop, rock, jazz, lofi, edm,
piano, guitar, drums, sax,
violin, synth, accordion,
happy, sad, energetic, calm

✓ 精煉到 5–8 個關鍵詞

RIGHT
indie folk, acoustic guitar,
light strings, soft male vocal,
melancholic and warm, 80 bpm

錯誤 5 · 全形標點符號

❌ 用了中文方括號

WRONG
【Verse】
街燈把影子拉長

✓ 一定要用半形 [ ]

RIGHT
[Verse]
街燈把影子拉長

Suno 只認得半形 [ ] ( )。輸入時注意把輸入法切到英文模式。
